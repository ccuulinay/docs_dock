# 银行营销活动模拟系统（Campaign Simulation System）
## ——基于因果数字孪生与大模型Agent的预演方案

---

## 1. 项目概述与核心目标

### 1.1 背景与灵感来源

本项目受 [Oransim](https://github.com/OranAi-Ltd/oransim)（Causal Digital Twin for Marketing at Scale）启发，将其"花钱之前先预演"的核心理念，从社交媒体广告投放场景迁移至**银行内部精准营销场景**。

Oransim 通过构建百万级虚拟消费者社会，结合结构因果模型（SCM）和 LLM 智能体，实现了在广告投放前对 ROI、传播曲线、用户情感的秒级预测。在银行场景中，我们不需要社交传播的级联扩散，但更需要**客户金融特征与营销响应之间的因果推理能力**，以及**多渠道触达策略的精准模拟**。

### 1.2 核心目标

在营销活动**设计阶段、实施之前**，输入拟定的活动方案（目标客群、选客策略、触达渠道、优惠力度、文案内容），系统能够预测：

| 预测维度 | 具体指标 |
|---------|---------|
| **响应预测** | 整体响应率、分层响应率、渠道响应率、响应时间分布 |
| **业务效果** | 预计产品销售额/AUM增长、新增客户数、交叉销售成功率 |
| **ROI预测** | 活动ROI、边际ROI、成本效益分析、盈亏平衡点 |
| **客群洞察** | 高响应人群画像、低响应人群特征、潜在流失风险 |
| **策略优化** | 最优选客策略、最优渠道组合、最优优惠力度、最优时机 |
| **反事实推演** | "如果换一批客户/换一种文案/换一个渠道，结果会怎样？" |

### 1.3 与现有方案的区别

| 维度 | 传统预测模型（如XGBoost） | 本方案（因果数字孪生 + LLM Agent） |
|------|----------------------|--------------------------------|
| 预测时机 | 基于历史数据的事后分析 | 活动实施前的实时模拟 |
| 可解释性 | 黑盒特征重要性 | 因果图可追溯每个预测节点的推导逻辑 |
| 反事实能力 | 无 | 支持 `do()` 操作，模拟策略变更效果 |
| 客户异质性 | 群体平均效应 | 个体级Agent模拟，支持CATE（条件平均处理效应） |
| 文案/创意评估 | 无法评估 | LLM Agent可模拟客户对具体文案的情感和决策 |
| 动态反馈 | 静态模型 | 支持多轮交互模拟（如客户犹豫、对比、咨询） |

### 3.3 关键数据加工

#### 3.3.1 客户行为序列嵌入（Behavioral Sequence Embedding）

将客户的行为日志转化为低维稠密向量，捕捉行为模式。关键行为模式包括：

- **储蓄增长型**：定期存款频次增加，AUM稳步上升
- **理财探索型**：频繁浏览理财页面，小额试投
- **信贷需求型**：查询贷款利率，信用卡额度使用率上升
- **流失预警型**：交易频次下降，AUM转移

#### 3.3.2 金融人格画像（Financial Persona）

基于聚类和业务规则，将客户划分为金融人格类型：

| 金融人格 | 特征描述 | 营销敏感度 |
|---------|---------|-----------|
| 稳健保值型 | 偏好存款、国债，风险厌恶 | 对收益率敏感，需要安全感 |
| 平衡配置型 | 股债混合，追求稳健增值 | 对专业建议信任度高 |
| 进取投资型 | 偏好权益类产品，能承受波动 | 对热点话题敏感 |
| 便利优先型 | 重视体验和便利性 | 对流程简化敏感 |
| 价格敏感型 | 关注手续费、优惠力度 | 对优惠活动高度敏感 |
| 关系驱动型 | 依赖客户经理建议 | 对人情关系敏感 |

## 4. 核心引擎详细设计

### 4.1 引擎1：客户数字孪生引擎（Customer Digital Twin Engine）

#### 4.1.1 设计目标

为银行客户群体构建高保真的虚拟副本，使得模拟结果能够反映真实客户群体的统计特性和行为模式。

#### 4.1.2 技术实现

**A. 基于IPF的人口校准（Iterative Proportional Fitting）**

参考 Oransim 的 IPF 方法，确保虚拟客户群体的分布与真实客户一致：

```
真实客群边际分布 --> IPF算法 --> 虚拟客群联合分布

约束维度：
  - 年龄 x 性别 x 地域
  - AUM等级 x 风险偏好
  - 产品持有数 x 数字化程度
  - 职业 x 收入等级
```

**B. 个体级客户状态向量（Customer State Vector）**

每个虚拟客户由以下状态向量描述：

```python
class CustomerState:
    # 基础属性 (静态)
    demographics = {}       # 人口统计特征
    financial_profile = {}  # 金融画像 (AUM, 收入, 风险偏好)
    product_ownership = {}  # 产品持有向量
    
    # 行为属性 (动态)
    behavioral_embedding = [] # 行为序列嵌入
    engagement_score = 0.0    # 互动活跃度
    channel_preference = []   # 渠道偏好分布
    
    # 营销状态 (动态)
    campaign_exposure_history = [] # 历史活动曝光记录
    response_history = []          # 历史响应记录
    fatigue_score = 0.0           # 营销疲劳度
    brand_affinity = 0.0          # 品牌/产品偏好度
    
    # 噪声项 (用于因果推断)
    idiosyncratic_noise = [] # 个体特异性噪声（Pearl abduction用）
```

**C. 营销疲劳度模型（Marketing Fatigue Model）**

银行客户被频繁营销会产生疲劳，影响响应率：

```
fatigue_score(t) = alpha * fatigue_score(t-1) + beta * exposure_count(t) + gamma * response_count(t)

其中：
  - alpha: 衰减系数（时间自然恢复）
  - beta: 曝光疲劳系数（每次曝光增加疲劳）
  - gamma: 响应恢复系数（响应后疲劳降低，正向反馈）
  
响应概率修正：
  P_respond_effective = P_respond_base * (1 - fatigue_penalty)
  fatigue_penalty = sigmoid(fatigue_score - threshold)
```

### 4.2 引擎2：世界模型（World Model / Response Prediction Model）

#### 4.2.1 设计目标

预测给定活动方案下，客户群体的响应概率、响应金额、响应时间等核心指标。

#### 4.2.2 模型架构

采用**因果Transformer世界模型**（参考 Oransim 的 CausalTransformerWorldModel），但适配银行特征：

输入特征分三类Token（CaT: Covariate-Arm-Treatment factorization）：

- **Covariate Tokens (客户特征)**：人口统计嵌入、金融特征嵌入、行为模式嵌入、渠道偏好嵌入
- **Treatment Tokens (活动策略)**：客群策略Token、触达策略Token、优惠策略Token、文案策略Token
- **Outcome Tokens (预测目标)**：响应概率、响应金额、响应时间、渠道选择、产品转化

架构组件：
1. **DAG-aware Attention**: 基于因果图限制注意力范围
2. **Per-arm Counterfactual Heads**: 每个策略arm一个预测头
3. **Representation Balancing**: HSIC/Adversarial-IPTW去偏
4. **Quantile Prediction**: P35/P50/P65分位数预测，提供置信区间

#### 4.2.3 训练策略

```python
# 训练数据构建
# 正样本：历史活动中响应的客户 (X, T, Y=1)
# 负样本：历史活动中未响应的客户 (X, T, Y=0)
# 反事实样本：通过倾向得分匹配(PSM)构建

class WorldModelTrainer:
    def train(self, campaign_history):
        # 1. 事实预测训练
        factual_loss = cross_entropy(Y_pred, Y_actual)
        
        # 2. 反事实预测训练（TARNet/Dragonnet）
        counterfactual_loss = mse(Y_cf_pred[T'], Y_observed[T'])
        
        # 3. 表示平衡约束
        balancing_loss = hsic(representation, treatment_assignment)
        
        # 4. 总损失
        total_loss = factual_loss + lambda1*cf_loss + lambda2*balancing_loss
```

#### 4.2.4 银行特有的预测目标

| 预测目标 | 说明 | 业务价值 |
|---------|------|---------|
| `response_prob` | 客户响应概率 | 预估触达转化率 |
| `response_value` | 预计响应金额（如存款金额、理财购买金额） | 预估AUM增长 |
| `response_time` | 预计响应时间分布 | 优化活动周期 |
| `channel_choice` | 客户偏好的响应渠道 | 优化渠道分配 |
| `product_uptake` | 产品购买概率 | 预估产品转化 |
| `aversion_risk` | 客户产生反感的概率 | 避免过度营销 |
| `churn_trigger` | 活动触发客户流失的概率 | 风险预警 |

#### 4.3.3 Pearl 三步骤反事实推理

```python
def counterfactual_predict(customer_state, campaign_design, intervention):
    """
    Pearl's 3-step counterfactual:
    1. Abduction: 基于观测数据推断噪声项
    2. Action: 施加干预 do(T=t')
    3. Prediction: 在新的因果路径上预测结果
    """
    # Step 1: Abduction - 推断个体噪声
    noise = infer_noise(customer_state, historical_outcomes)
    
    # Step 2: Action - 施加干预
    modified_campaign = apply_intervention(campaign_design, intervention)
    # e.g., intervention = {"do(channel": "sms->app_push")", "do(discount": "5%->8%")"}
    
    # Step 3: Prediction - 在修改后的SCM上向前传播
    predicted_outcome = forward_scm(customer_state, modified_campaign, noise)
    
    return predicted_outcome
```

| **渠道变更** | "如果增加客户经理面访比例到30%，高端客户响应率会提升多少？" | 渠道策略优化 |

| **文案变更** | "如果强调稳健收益而非高收益，风险厌恶型客户的响应率如何变化？" | 内容策略优化 |

### 4.4 引擎4：LLM Agent 模拟引擎（Bank Customer Agent）

#### 4.4.1 设计目标

利用大模型的推理和生成能力，模拟真实客户对营销活动的**认知、情感、决策**过程，弥补纯统计模型在"理解文案"和"模拟决策心理"方面的不足。

#### 4.4.2 Agent架构

```python
class BankCustomerAgent:
    """银行客户智能体，基于LLM模拟客户决策"""
    
    # 人格基础（来自数字孪生）
    persona_card = ""       # 人格卡片（自然语言描述）
    financial_profile = {}  # 金融画像
    behavioral_pattern = "" # 行为模式摘要
    historical_context = "" # 历史交互摘要
    
    # LLM配置
    llm_model = "gpt-4o"   # 或其他内部部署模型
    temperature = 0.3       # 低温度保证一致性
    
    def evaluate_campaign(self, campaign):
        """评估营销活动，返回决策结果"""
        prompt = f"""
        你是一位具有以下特征的客户：
        {self.persona_card}
        
        你的金融状况：
        {self.financial_profile}
        
        你最近的行为模式：
        {self.behavioral_pattern}
        
        你现在收到了以下营销信息：
        渠道：{campaign.channel}
        文案：{campaign.copy}
        优惠：{campaign.offer}
        产品：{campaign.product}
        
        请基于你的人格特征和金融状况，做出真实反应：
        1. 你会注意这条消息吗？（注意力）
        2. 你的第一反应是什么？（情感）
        3. 你会考虑采取行动吗？（意向）
        4. 如果考虑，你的顾虑是什么？（障碍）
        5. 最终决定：响应/忽略/反感（决策）
        6. 如果响应，预计投入金额？（金额）
        7. 决策理由（解释）
        
        以JSON格式输出。
        """
        response = self.llm.complete(prompt)
        return parse_agent_response(response)
```

#### 4.4.3 人格卡片生成（Persona Card Generation）

从结构化特征自动生成自然语言人格描述：

```python
def generate_persona_card(customer_state):
    """将客户状态向量转换为LLM可理解的人格描述"""
    
    template = """

    - 每月交易频次：{transaction_freq}
    - 主要使用渠道：{primary_channel}
    - 数字化程度：{digital_level}
    - 对银行服务的依赖度：{dependency}
    
    你的性格倾向：
    - 决策风格：{decision_style}
    - 对优惠敏感度：{offer_sensitivity}
    - 对专业建议信任度：{advice_trust}
    - 营销疲劳度：{fatigue_level}
    
    注意：请始终保持这个人设，做出符合该人设的决策。
```

#### 4.4.4 两阶段模拟策略

为平衡成本与效果，采用**统计Agent + LLM Agent**的两阶段策略：

```
Phase 1: 统计筛选（Statistical Screening）
  +-- 使用世界模型对所有虚拟客户进行快速评分
  +-- 筛选出高不确定性、高业务价值的"关键客户"
  +-- 例如：前20%高响应概率 + 后20%低响应概率 + 随机采样
  
Phase 2: LLM深度模拟（LLM Deep Simulation）
  +-- 仅对Phase 1筛选出的关键客户启动LLM Agent
  +-- 每个Agent模拟完整的认知-情感-决策过程
  +-- 输出详细的决策理由和金额估计
  
Phase 3: 结果聚合（Aggregation）
  +-- 统计Agent结果：整体响应率、金额分布
  +-- LLM Agent结果：定性洞察、决策理由聚类
  +-- 融合输出：量化预测 + 定性解释
```

**成本控制**：
- 统计Agent：零LLM成本，毫秒级
- LLM Agent：仅模拟关键客户（如1万客群中的500人），可控成本
- 支持 `use_llm=False` 的纯统计模式用于快速迭代

### 4.5 引擎5：财务ROI计算引擎（Banking ROI Engine）

#### 4.5.1 银行营销ROI的特殊性

银行营销ROI不同于电商的直接销售额，需要综合考虑：

```
总收益 = 直接产品收益 + AUM增长收益 + 交叉销售收益 + 长期客户价值收益

直接产品收益：
  - 存款产品：利息差收益（AUM x 利差 x 持有期）
  - 理财产品：申购费/管理费分成
  - 信贷产品：利息收入
  - 保险产品：佣金收入

AUM增长收益：
  - 新增AUM x 综合收益率 x 预计留存期
  
成本 = 触达成本 + 优惠成本 + 人力成本 + 系统成本
  - 触达成本：短信/App Push/电销/面访的单位成本 x 触达量
  - 优惠成本：利率上浮/积分/礼品/返现的直接成本
  - 人力成本：客户经理时间成本
  - 系统成本：活动配置、数据加工、效果追踪的IT成本
```

#### 4.5.2 ROI计算模型

```python
class BankingROIEngine:
    def calculate_roi(self, campaign, predictions):
        # 收入侧
        direct_revenue = predictions.product_sales * campaign.product_margin
        aum_revenue = predictions.aum_increase * self.aum_yield_rate * self.aum_retention_period
        cross_sell_revenue = predictions.cross_sell_uptake * self.cross_sell_value
        ltv_increment = predictions.new_customer_count * self.new_customer_ltv
        
        total_revenue = direct_revenue + aum_revenue + cross_sell_revenue + ltv_increment
        
        # 成本侧
        touch_cost = campaign.touch_volume * campaign.channel_unit_cost
        offer_cost = predictions.respondent_count * campaign.offer_cost_per_head
        labor_cost = campaign.required_labor_hours * self.labor_hour_rate
        system_cost = campaign.system_cost
        
        total_cost = touch_cost + offer_cost + labor_cost + system_cost
        
        # ROI计算
        roi = (total_revenue - total_cost) / total_cost
        marginal_roi = self.calculate_marginal_roi(campaign, predictions)
        
        return ROISummary(
            total_revenue=total_revenue,
            total_cost=total_cost,
            roi=roi,
            marginal_roi=marginal_roi,
            break_even_response_rate=total_cost / (total_revenue / predictions.respondent_count)
        )
```

### 5.1 用户界面架构

系统界面分为三大面板：

**面板1：活动设计画布**
- 目标产品选择
- 客群选择器（规则圈选/上传名单/相似人群扩展）

- 优惠配置（利率上浮 / 积分奖励 / 费率折扣）

**面板2：预测与反事实面板**
- KPI预测卡（响应率、响应人数、AUM增长、ROI及置信区间）
- 反事实沙盒（原方案 vs 方案A vs 方案B，拖动滑块实时对比）
- 客群响应热力图（按AUM/年龄/地域的响应率分布）

**面板3：可解释性与审计面板**
- 因果路径图（各因素对预测结果的贡献分解）
- LLM Agent洞察（典型客户反馈聚类：积极型/犹豫型/消极型）
- 策略建议（系统自动生成的优化建议）

### 5.2 核心交互流程

```
用户旅程：

Step 1: 活动设计
  +-- 选择目标产品（如"大额存单"）
  +-- 定义目标客群（规则圈选/上传名单/相似人群扩展）
  +-- 配置触达策略（渠道组合、频次、时机）
  +-- 设置优惠方案（利率上浮、积分、礼品）
  +-- 编辑营销文案（支持AI生成+人工审核）
  +-- 设定预算约束
  
Step 2: 一键模拟
  +-- 系统生成虚拟客群（10万级客户数字孪生）
  +-- 世界模型预测基础响应指标
  +-- LLM Agent模拟关键客户决策（可选）
  +-- 因果引擎计算反事实场景
  +-- ROI引擎计算财务指标
  
Step 3: 策略优化
  +-- 查看预测结果与置信区间
  +-- 对比多个策略方案（A/B/C）
  +-- 查看可解释性分析（因果路径分解）
  +-- 查看LLM Agent的客户洞察
  +-- 调整参数，重新模拟
  
Step 4: 决策输出
  +-- 导出最终活动方案
  +-- 生成预测报告（含风险提示）
  +-- 推送至活动执行系统
  +-- 设定实际追踪指标，用于模型校准
```

## 9. 总结

本方案借鉴 Oransim 的**因果数字孪生**理念，将其从社交媒体广告投放场景迁移至**银行内部精准营销场景**，构建了一个在营销活动实施前即可预测响应结果和ROI的模拟系统。

**核心创新点**：
1. **银行客户数字孪生**：基于IPF校准和金融人格画像，构建高保真虚拟客户群体
2. **因果世界模型**：采用因果Transformer架构，支持反事实推理和策略优化
3. **LLM客户Agent**：模拟真实客户的金融决策心理，提供定性洞察
4. **银行专属ROI引擎**：综合考虑AUM增长、产品收益、客户价值等银行特有指标

- 在活动实施前识别最优策略组合，降低试错成本
