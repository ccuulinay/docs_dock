# OpenClaw 智能调研报告系统 - 详细执行计划

## 1. 项目概述

### 1.1 项目目标
构建一个基于 OpenClaw 的自动化调研报告生成系统，允许 CEO 通过 iMessage 发送调研主题，系统自动完成深度研究、报告生成、网页化转换，并通过邮件推送可交互的网页报告。

### 1.2 核心价值
- **随时随地发起调研**：通过 iMessage 移动端触发
- **端到端自动化**：从主题输入到报告交付全程自动化
- **专业级交付物**：可交互的网页版报告替代传统文档
- **信息可追溯**：保留研究过程和数据来源

### Phase 4: 幻觉控制与质量保障（第4-6周）

#### 3.8 大模型幻觉控制体系

**多层防御策略：**

**第一层：检索增强（RAG）**
- 强制要求每个结论必须附带来源引用
- 实现引用溯源链路（结论 → 原文片段 → 数据源URL）
- 使用重排序模型（Reranker）提升检索精度

**第二层：事实核查链**
- 独立 `fact_checker` Skill 进行交叉验证
- 关键数据点需至少2个独立源确认
- 置信度评分：<0.7分的数据标记为"待核实"

**第三层：专家审核模式**
- 高价值报告启用"慢思考模式"（多轮Self-Reflection）
- 实现 Chain-of-Verification 机制
- 敏感结论添加免责声明

**第四层：人机回环（HITL）**
- 设置置信度阈值，低于阈值时标记人工复核
- CEO可一键触发深度核查流程

**技术实现：**
```python
class HallucinationGuard:
    def verify_claim(self, claim, sources):
        # 1. 提取可验证的命题
        verifiable_facts = self.extract_facts(claim)

        # 2. 多源交叉验证
        for fact in verifiable_facts:
            evidence = self.multi_source_search(fact)
            consensus_score = self.calculate_consensus(evidence)

        # 3. 置信度计算
        confidence = self.weighted_score(consensus_score, source_quality)

        # 4. 标记低置信度内容
        if confidence < 0.7:
            return {"verified": False, "flag": "NEEDS_REVIEW"}
```

**排版规范：**

- 表格规范：左对齐文本，右对齐数字，保留2位小数
- 图表要求：所有图表必须有标题、数据来源标注、图例说明

## 5. 风险识别与应对

### 5.1 技术风险

| 风险 | 影响 | 可能性 | 应对策略 |
|------|------|--------|----------|
| iMessage自动化不稳定 | 高 | 中 | 开发备用方案（Telegram/WhatsApp）；部署macOS Mini本地监听 |
| MCP服务大规模失效 | 高 | 低 | 每个数据类型至少2个备用源；本地缓存最近30天数据 |
| 大模型幻觉导致错误结论 | 极高 | 中 | 强制引用要求；置信度评分<0.7时人工复核；购买专业责任保险 |
| 报告生成时间过长 | 中 | 中 | 实现流式生成（先出摘要）；异步任务+进度通知；优化并行Skill执行 |

### 5.2 业务风险

| 风险 | 应对策略 |
|------|----------|
| CEO对报告质量不满意 | 前3次报告人工预审；建立快速反馈迭代机制 |
| 敏感数据泄露 | 本地部署优先；传输加密；报告访问权限控制（密码/IP限制） |
| 数据源版权争议 | 仅引用公开数据/购买商业数据许可；避免大段原文复制 |

### 5.3 缓解措施

**质量保证：**
