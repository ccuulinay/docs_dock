# 舆情分析 Workflow Prompt

## 概述
本workflow用于对指定地区(area)、指定主题(topic)的舆情进行系统性分析，帮助追踪政策动向、市场趋势和关键事件，并输出可行动的洞察报告。

### 输入参数
| 参数 | 说明 | 示例 |
|------|------|------|
| `<area>` | 目标地区 | 香港、EU/US、Global |
| `<topic>` | 分析主题 | 金融科技、AI监管、银行业 |
| `<days>` | 回溯天数 | 默认7天 |

---

## Step 1: 信息源精选

### 确定数据源文件
根据给定的`<area>`，选择对应的信息源文件`<news_source_file>`：
- EU/US → `news_source_EU_US.md`
- 香港/澳门 → `news_source_HK_MAU.md`
- 中国大陆 → `news_source_MLD.md`
- 其他 → 结合上述三个文件

### 确定精选数据源类型
从以下7类数据源中，根据`<topic>`选择**3类**作为精选源`<news_source_type>`：

| 编号 | 数据源类型 | 适用场景 |
|------|------------|----------|
| 1 | 官方政策与监管 | 政策动向、监管变化 |
| 2 | 官方数据发布 | 经济指标、市场数据 |
| 3 | 交易所与市场基础设施 | 公司财报、上市披露 |
| 4 | 上市公司与机构原始披露 | 企业公告、财报 |
| 5 | 权威媒体与官媒 | 新闻报道、市场解读 |
| 6 | 行业协会与研究机构 | 行业报告、研究 |
| 7 | 重点科技企业官网 | 企业动态、产品 |

### 精选原则
- 优先级：1类 > 5类 > 3类 > 2类 > 6类 > 7类
- 确保覆盖一手来源（官方）与二手解读（媒体）

---

## Step 2: 热点新闻清单采集

### 采集任务
从选定的3类精选数据源`<news_source_type>`，结合指定的`<topic>`，对过去`<days>`日的新闻进行搜索，生成**15条**相关度最高的新闻清单。

### 新闻清单采集字段
| 字段 | 说明 |
|------|------|
| 日期 | YYYY-MM-DD |
| 来源名称 | 具体媒体/机构名称 |
| 来源类型 | 对应Step 1的编号 |
| 主题 | 所属细分主题 |
| 标题 | 新闻标题 |
| 摘要 | 100字内核心要点 |
| 关键词 | 3-5个关键词 |
| 相关度 | High/Medium/Low |
| 原始链接 | 可访问的URL |

### 输出格式
```markdown
## 新闻清单 (<area> - <topic> - 最近<days>天)

| # | 日期 | 来源 | 标题 | 相关度 | 关键词 |
|---|------|------|------|--------|--------|
| 1 |      |      |      |        |        |
```

---

## Step 3: 详细新闻信息搜集与情感分析

### 筛选与深度分析
从`<news_topic_last7d>`清单中，选取**相关度最高的前10条**，进行深度内容搜集。

### 分析框架
针对每条新闻，从以下维度进行分析：

1. **事件核心**：What happened? (50字内)
2. **影响主体**：Who/Which sector affected?
3. **影响方向**：Positive / Negative / Neutral
4. **情感强度**：Strong / Moderate / Weak
5. **对本地区的启示**：结合`<area>`的具体影响
6. **后续关注点**：需要追踪的事项

### 归纳总结
根据`<topic>`和`<area>`，生成归纳总结，输出文件`<news_summary_topic_last7d>`：

```markdown
# <area> <topic> 舆情分析报告 (最近<days>天)

## 核心要点
- [要点1]
- [要点2]
- [要点3]

## 重大事件分析

### 1. [事件标题]
- **日期**: YYYY-MM-DD
- **来源**: [来源名称]
- **事件核心**: [核心描述]
- **影响主体**: [行业/地区]
- **情感判断**: [Positive/Negative/Neutral] - [强度]
- **地区启示**: [对<area>的影响分析]
- **后续关注**: [待追踪事项]


---

## 执行要点

1. **Step 1**: 严格遵守`<news_source_file>`选择对应数据源
2. **Step 2**: 采集字段如无法获取，标注"待确认"而非留空
3. **Step 3**: 无法获取原文内容的新闻，跳过但保留在清单中
4. **Step 4**: 基于有限信息仍需给出合理判断，避免过度犹豫
5. **质量控制**: 
   - 每条High相关度新闻必须包含情感分析
   - 每份报告必须包含至少1条actionable insight
   - 引用数据需注明来源和时间

### 交付物
- `<news_topic_last7d>`: 新闻清单MD文件
- `<news_summary_topic_last7d>`: 归纳总结MD文件
